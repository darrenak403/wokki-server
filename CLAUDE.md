# Wokki Server — Claude Code

@AGENTS.md

Branch workspace rule lives in AGENTS.md: selected branch URL scopes workspace/sidebar actions; Admin creates staff through `/employees` (User + Employee together; same-org legacy orphan Users linked there) with department → auto Active LocationMembership; existing staff User↔Manager via `POST /employees/{id}/role-transition` (not `PATCH /users/{id}/role`); parallel org join via `/auth/register-employee` + `/org-join-requests` (BR-020). No standalone `/users` staff. Manager only `LocationManager`-assigned locations. Org package gate: register creates org pending package; PlatformOperator activates/renews via `/api/v1/platform/organizations/{id}/subscription`, otherwise org users get `ORG_PACKAGE_NOT_ACTIVATED` / `ORG_PACKAGE_EXPIRED`; every subscription change must write immutable ledger + `AuditLog` in the same transaction. Platform support console phase 1 has no impersonation and no tenant business-data edits; active org analytics are only login, schedule publish/suggest/apply, attendance clock-in/out, or chat message events. Schedule suggestions apply by exact `(shiftDefinitionId, employeeId, date)` tuple. Ephemeral per-call hints via `/insights/chat?HintMode=true` (Bedrock advisory): 3 shapes (PreferenceWeight, TempMinMax, AvoidPairing), passed to `suggest` only, never persisted; `apply-suggestions` has no hint field (BR-097). Shift swap marketplace: Draft-only feed at `/api/v1/swap-posts`, FCFS accept, `GET /self/schedule/draft/{week}/assignments` for picker; publish hides pending posts; legacy `/api/v1/swap-requests` removed. FE: `user/swap` (`SwapMarketplacePanel`), admin/manager audit (`SwapAuditPanel`). Attendance verification is advisory: optional geo/IP/photo check-in flags (BR-098), never block, face enrollment on first photo per employee, cleanup after 30 days (BR-098).
